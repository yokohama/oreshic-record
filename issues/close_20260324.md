# --catフラグを追加

現在以下の使用になっている

| 対象リソース | フラグ | 範囲 |
|--|--|--|
| Command / Track | --run | セクションの中のコマンドを実行 |
| Writeup | --run | - |
| Command / Track | --open | ファイルをeditorで開き、セクションにジャンプ |
| Writeup | --open | ファイルをeditorでオープン |
| Command / Track | --del | 対象のセクションを削除 |
| Writeup | --del | ファイルを削除 |

## 何が問題か？

既存の仕様だと、 `ors serach query 4` 等としたときに、必ずMD&ページャーで開かれるため、テキストとして扱いずらい。
`--cat`を追加して、生テキストの出力を実装する。

以下のフラグを追加する

| 対象リソース | フラグ | 範囲 |
|--|--|--|
| Command / Track | --cat | ファイル全体ではなく、該当セクションをcat |
| Writeup | --cat | ファイルをcat |


## 追加されるコマンドの組み合わせ

```
ors search command 3 --cat
ors search track 3 --cat
ors search writeup 3 --cat
ors search query hoge 3 --cat
```

## 補足
既存の、`--run`、`--open` 等に合わせてショートは作成しない。
